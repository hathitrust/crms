Note: you can use an asterisk (*) in a search field as a wildcard that will match zero or more characters.
Example: searching for volume id wu.89* will match wu.89090507310, wu.89092821453, etc....
Example: searching for title *trigonometric* will match "Natural trigonometric functions to seven decimal places for every ten seconds of arc"


Note: you can use the <, <=, >, and >= operators before a numeric expression.
Example: searching for pub date <=1923 will retrieve all volumes published before or in 1923.


Note: OR takes precedence over AND and NOT when you use all three search fields. So x AND y OR z is interpreted as x AND (y OR z).


  • Searching by review is the search functionality from version 1.0 of the system -- volumes will be grouped together only when ordering by Identifier.
  • Searching by volume retrieves volumes for which any single review matches the search criteria; ordering/sorting is done on the review that meets the search criteria, not the additional ones.
  • Searching by volume, and clicking on the "Search across reviews (wide)" checkbox does a wide search: in which volumes are retrieved if any of the reviews match; ordering/sorting is done on the minimum or maximum values for the sort term, of the reviews that meet the search criteria.
  • Searching by volume always keeps reviews for the same volume together.
  • Examples:
    • To search for all volumes for which one or more of its reviews is pd/ren: use narrow search.
    • To search for all volumes whach have a review by Anne and another review by Greg: use wide search.
